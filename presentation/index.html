SO WHAT IS THE MANDELBAUM?

🌳

In 1980, a man named Benoît Mandelbrot looked at the equation z → z² + c and thought:

"what if I just... loop this forever"

(he was a genius. we are just vibing.)

THE TERRIFYING FORMULA

z_(n+1) = z_n² + c where: z = complex number (scary) c = also complex (also scary) n = loop counter ∞ = when you give up

If |z| stays ≤ 2 forever → point is IN the set → black
If it escapes to ∞ → colour by how fast it bolted

That's literally it. The whole thing. I promise.

NOW WHAT IS KUBERNETES?

🤔

Imagine you have a bunch of computers.
Now imagine you need something to babysit all those computers.
Now imagine the babysitter is also made of computers.

That's Kubernetes.

Greek for "helmsman". The logo is a steering wheel.
Everyone calls it k8s because typing is hard.

HOW WE MADE IT NOT TERRIBLE 🚀

  • NumPy vectorised math
    whole rows at once, not pixel by pixel
  • Numba JIT compiler
    inner loop compiled to native CPU code
  • parallel rows (prange)
    all CPU cores crunch simultaneously
  • float64 precision
    deep zoom without pixel soup
  • fast PNG (compress level 1)
    small file, not slow to encode
  • client-side pan & zoom
    canvas transform, zero server trips

result: from "make coffee while it loads" → "actually usable"

THE YAML KINGDOM 📜

Kubernetes is controlled entirely through YAML files.
Yes. All of it. It's YAML all the way down.

deployment.yaml → "run 3 of this container" → restart if it crashes → CPU / RAM limits service.yaml → one stable address → load-balances across pods → pods come & go, IP stays
ingress.yaml → routes the outside world in → / goes to the coordinator → (Traefik handles it) rbac.yaml → who is allowed to do what → coordinator gets read access → so it can list pods for the UI

kubectl apply -f k8s/ — and the cluster figures out the rest

THE UGLY TRUTH 😬

We split one render across 3 pods → great for one user.
But what happens when 3 users open the site at the same time?

👤 User 1
👤 User 2
👤 User 3
→
👑 Coord
→
😵 Pod 1 (busy)
😵 Pod 2 (busy)
😵 Pod 3 (busy)

All 3 users share the same 3 pods. Requests queue. No parallel speedup.
Real fix: auto-scale pods based on load (k8s HPA) — but that's a problem for future us 🫡
